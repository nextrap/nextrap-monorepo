# @nextrap/style-base

## Purpose

Provides the global Nextrap design tokens and container-related CSS variables.

Use this package to bootstrap theme variables such as colors, typography, spacing, border radii, and container widths for the main document.

This package is a **global theme/token layer**. It is not a component stylesheet.

## Important rule

- Include `@nextrap/style-base` / `@nextrap/style-base/module` only in the **main document / theme stylesheet**.
- Do **not** include it inside the **shadow DOM** styles of components.
- Add it once at theme level, then consume the generated CSS variables in components and theme partials.

## Main API

### Theme mixin

Preferred SCSS API:

```scss
@use '@nextrap/style-base/module' as nextrapBase;

.theme-example {
  @include nextrapBase.nextrap-theme(
    (
      primary: #2b94d6,
      accent: #2b94d6,
      secondary: #4c4c4c,
      font-family-base: sans-serif,
      rcs: true
    )
  );
}
```

The mixin generates the core CSS custom properties and container tools for the given theme scope.

## Typical usage in this project

Use it in theme entry files, for example:

```scss
@use '@nextrap/style-base/module' as nextrapBase;

.theme-osman {
  @include nextrapBase.nextrap-theme($theme, (
    primary: #2b94d6,
    accent: #2b94d6,
    secondary: #4c4c4c,
    gray: #6f6f6f,
    enableFontConfig: true,
    rcs: true
  ));
}
```

After that, use the generated variables in other styles:

```scss
color: var(--nt-primary);
background: var(--nt-body);
border-radius: var(--nt-border-radius);
max-width: var(--nt-container-width);
gap: var(--nt-gap);
```

## What the theme map can override

Common overrides include:

- `primary`, `accent`, `secondary`, `tertiary`
- `success`, `info`, `warning`, `danger`
- `light`, `dark`, `white`, `black`, `gray`
- `text`, `header`, `link`, `link-on-dark`
- `font-family-base`, `font-family-header`
- `font-size-base`, `font-size-header`
- `font-weight-base`, `font-weight-header`, `font-weight-bold`
- `line-height-base`, `line-height-header`
- `border-radius`, `border-radius-lg`, `border-radius-sm`
- `gap`, `text-gap`, `space`, `content-space`
- `space-scale`, `text-scales`, `header-scales`
- `enableFontConfig`, `enableScrollOffset`, `rcs`

## Behavior notes

- If only `accent` is provided, `primary` is derived from it.
- If only `primary` is provided, `accent` is derived from it.
- Subtle/emphasis variants are generated automatically.
- Container-related helpers are included by the mixin.

## CSS variables reference

### Core color variables

- `--nt-primary`: primary brand color.
- `--nt-primary-subtle`: lighter primary variant for soft backgrounds.
- `--nt-primary-emphasis`: stronger/darker primary variant.
- `--nt-secondary`: secondary theme color.
- `--nt-secondary-subtle`: lighter secondary variant.
- `--nt-secondary-emphasis`: stronger/darker secondary variant.
- `--nt-tertiary`: tertiary theme color.
- `--nt-tertiary-subtle`: lighter tertiary variant.
- `--nt-tertiary-emphasis`: stronger/darker tertiary variant.
- `--nt-accent`: accent theme color.
- `--nt-accent-subtle`: lighter accent variant.
- `--nt-accent-emphasis`: stronger/darker accent variant.
- `--nt-success`: success/positive state color.
- `--nt-success-subtle`: lighter success variant.
- `--nt-success-emphasis`: stronger/darker success variant.
- `--nt-info`: informational state color.
- `--nt-info-subtle`: lighter info variant.
- `--nt-info-emphasis`: stronger/darker info variant.
- `--nt-warning`: warning state color.
- `--nt-warning-subtle`: lighter warning variant.
- `--nt-warning-emphasis`: stronger/darker warning variant.
- `--nt-danger`: danger/error state color.
- `--nt-danger-subtle`: lighter danger variant.
- `--nt-danger-emphasis`: stronger/darker danger variant.
- `--nt-light`: light neutral color.
- `--nt-light-subtle`: very light variant of `--nt-light`.
- `--nt-light-emphasis`: darker/emphasized light variant.
- `--nt-dark`: dark neutral color.
- `--nt-dark-subtle`: softer dark variant.
- `--nt-dark-emphasis`: stronger/darker dark variant.
- `--nt-white`: white color token.
- `--nt-black`: black color token.
- `--nt-gray`: base gray token.

### Gray scale variables

These are generated from `--nt-gray`:

- `--nt-gray-100`: lightest gray.
- `--nt-gray-200`: very light gray.
- `--nt-gray-300`: light gray.
- `--nt-gray-400`: soft gray.
- `--nt-gray-500`: medium-light gray.
- `--nt-gray-600`: base gray level.
- `--nt-gray-700`: dark gray.
- `--nt-gray-800`: very dark gray.
- `--nt-gray-900`: darkest gray.
- `--nt-gray-dark`: named dark gray helper.

### Surface and body variables

- `--nt-body`: default page/background surface color.
- `--nt-body-secondary`: secondary body/surface color.
- `--nt-body-tertiary`: tertiary body/surface color.
- `--nt-background`: default background token, currently mapped to white.

### Text and link variables

- `--nt-text`: default body text color.
- `--nt-header`: default heading color.
- `--nt-link`: default link color.
- `--nt-link-on-dark`: link color for dark backgrounds.
- `--nt-text-decoration-link`: default text decoration for links.
- `--nt-text-on-primary`: readable text color for primary backgrounds.
- `--nt-text-on-secondary`: readable text color for secondary backgrounds.
- `--nt-text-on-tertiary`: readable text color for tertiary backgrounds.
- `--nt-text-on-accent`: readable text color for accent backgrounds.
- `--nt-text-on-success`: readable text color for success backgrounds.
- `--nt-text-on-info`: readable text color for info backgrounds.
- `--nt-text-on-warning`: readable text color for warning backgrounds.
- `--nt-text-on-danger`: readable text color for danger backgrounds.
- `--nt-text-on-light`: readable text color for light backgrounds.
- `--nt-text-on-dark`: readable text color for dark backgrounds.
- `--nt-text-on-white`: readable text color for white backgrounds.
- `--nt-text-on-black`: readable text color for black backgrounds.

### Typography variables

- `--nt-font-family-base`: base/body font family.
- `--nt-font-family-header`: heading font family.
- `--nt-font-size-base`: base text size.
- `--nt-font-size-header`: maximum heading size reference.
- `--nt-font-size-small`: reduced text size.
- `--nt-font-weight-base`: default body font weight.
- `--nt-font-weight-header`: default heading font weight.
- `--nt-font-weight-bold`: shared strong/bold weight token.
- `--nt-line-height-base`: default body line height.
- `--nt-line-height-header`: default heading line height.
- `--nt-font-size-root`: root responsive size token used as rem sizing basis; `clamp(...)` when `rcs: true`, otherwise `1rem`.

### Font scale variables

- `--nt-fs-1` to `--nt-fs-6`: generic text scale derived from `--nt-font-size-base` and `text-scales`.
- `--nt-fs-h1` to `--nt-fs-h6`: heading sizes interpolated between `--nt-font-size-header` and `--nt-font-size-base`.

### Spacing variables

- `--nt-gap`: general layout gap token.
- `--nt-text-gap`: spacing token for text blocks.
- `--nt-default-gap-x`: default horizontal padding helper.
- `--nt-default-gap-y`: default vertical padding helper.
- `--nt-space`: base spacing unit.
- `--nt-space-1` to `--nt-space-5`: scaled spacing values derived from `space-scale`.
- `--nt-content-space`: larger content spacing base unit.
- `--nt-content-space-1` to `--nt-content-space-5`: scaled content spacing values derived from `space-scale`.

### Border variables

- `--nt-border-radius`: default border radius token.
- `--nt-border-radius-sm`: small border radius token.
- `--nt-border-radius-xl`: large border radius token.
- `--nt-border-width`: default border width token.

### Container and scrolling variables

- `--nt-container-width`: responsive container width token set by breakpoints.
- `--nt-scroll-padding-top`: default scroll offset helper, based on viewport height.

Current container widths set by media queries:

- `>= 576px`: `--nt-container-width: 540px`
- `>= 768px`: `--nt-container-width: 720px`
- `>= 992px`: `--nt-container-width: 960px`
- `>= 1200px`: `--nt-container-width: 1140px`
- `>= 1400px`: `--nt-container-width: 1320px`

## Agent hints

- Prefer existing theme variables like `var(--nt-primary)`, `var(--nt-body)`, `var(--nt-gap)`, and `var(--nt-container-width)` instead of hardcoding repeated values.
- When modifying a theme, adjust the `nextrap-theme(...)` input first before introducing many one-off overrides.
- For component shadow DOM styling, use package-specific reset/base patterns, not `@nextrap/style-base` directly.
- Keep theme setup centralized in theme entry files such as `theme/<name>/_theme.scss`.
