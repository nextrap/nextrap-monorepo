# @nextrap/style-utils

## Purpose

Provides reusable Nextrap utility classes for styling individual elements directly in markup.

Use this package when HTML or Markdown content should be styled by attaching utility classes to the specific item that needs the styling instead of introducing custom one-off CSS classes.

## Important rule

**These styles should be used in HTML and Markdown files to individually style items.**

Prefer applying the provided utility classes directly on the target element, for example on:

- lists
- text blocks
- containers
- rows and columns
- borders
- spacing wrappers
- display or flex wrappers
- tables

Do not create new ad-hoc helper classes when an existing utility class from `@nextrap/style-utils` already fits.

## Import

```ts
import '@nextrap/style-utils';
```

Or in SCSS:

```scss
@use '@nextrap/style-utils';
```

## Typical usage

This package is intended for utility-class usage in markup.

### HTML example

```html
<ul class="list-unstyled">
  <li class="text-primary mb-2">Item 1</li>
  <li class="text-success border-1 border-dark p-2">Item 2</li>
</ul>

<div class="row g-2">
  <div class="col-4">Side</div>
  <div class="col-8">Main</div>
</div>
```

### Markdown example

Use utility classes in Markdown where attribute/class syntax or embedded HTML is supported:

```md
- Item 1
- Item 2
- Item 3
{: .list-unstyled }

## Note
{: .text-primary .mb-2 }

<div class="border-1 border-dark p-2">Highlighted content</div>
```

- https://getbootstrap.com/docs/5.3/utilities/api/
- https://getbootstrap.com/docs/5.3/utilities/spacing/

This package follows Bootstrap-style utility usage where it makes sense, but it is **not a full Bootstrap clone**. Use the following classes that are actually provided by `@nextrap/style-utils`.

## Agent hints

- Prefer these utility classes when styling single items in HTML or Markdown content.
- Start with existing utility classes before adding new custom selectors.
- For content-driven styling, keep the styling close to the item by placing classes directly on the element.
- For Markdown-rendered content, prefer supported class/attribute syntax or embedded HTML over adding page-specific CSS.
- If repeated combinations appear often, first check whether the combination should stay as utilities or should become a shared semantic component/style in another package.
