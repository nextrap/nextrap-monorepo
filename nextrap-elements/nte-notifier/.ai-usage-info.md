# @nextrap/nte-notifier

## Purpose

Global notifier dialog for Nextrap applications.

It is meant to be mounted once on the page and then controlled through global `window` custom events with the `nextrap:` prefix.

## Mounting

Mount it once, usually directly in `body`:

```html
<body>
  <nte-notifier></nte-notifier>
</body>
```

Do not use it as a wrapper around page content.

## Construction rule

This is an `nte-*` element and should follow the shared `@nextrap/nte-core` conventions.
The component itself uses `nextrap_element({ eventBinding: true })` and `@Listen(..., { target: 'window' })` for global events.

## Event model

Supported global events:

- `nextrap:loading`
- `nextrap:progress`
- `nextrap:success`
- `nextrap:fail`
- `nextrap:info`
- `nextrap:confirm`

## Common usage examples

### Progress / wait dialog

```ts
window.dispatchEvent(
  new CustomEvent('nextrap:progress', {
    detail: {
      title: 'Datei-Upload',
      progress: 45,
      message: 'Dateien werden verarbeitet...',
      cancelable: true,
    },
  }),
);
```

If `title` is omitted, the notifier falls back to the default status title.

### Error dialog with expandable details

```ts
window.dispatchEvent(
  new CustomEvent('nextrap:fail', {
    detail: {
      message: 'Import fehlgeschlagen.',
      details: 'Zeile 152: Ungültiges Datumsformat in Spalte "deliveryDate".',
      autoClose: false,
      cancelable: true,
    },
  }),
);
```

### Confirm dialog

```ts
window.dispatchEvent(
  new CustomEvent('nextrap:confirm', {
    detail: {
      message: 'Wirklich löschen?',
      cancelable: true,
      actions: [
        {
          label: 'Löschen',
          variant: 'danger',
          callback: () => console.log('confirmed'),
        },
      ],
    },
  }),
);
```

## Accessibility behavior

- The notifier renders a native `<dialog>`.
- Loading and progress states are announced as wait dialogs for screen readers.
- Confirm states are announced as prompt / confirmation dialogs.
- Error details are rendered in a native `<details><summary></summary></details>` block.
- If `cancelable: true` is set, the dialog shows a close button (`×`) and reacts to ESC / cancel.
