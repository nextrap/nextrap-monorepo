# @nextrap/nte-core

## Purpose

Core factory for all `nte-*` web components.

Use this package when creating a Nextrap element that extends Lit and should get the shared Nextrap / TrunkJS browser mixins in a consistent way.

## Construction rule

For AI-generated code and package conventions:
- all `nte-*` elements should be constructed via `nte-element()`
- in the current codebase this corresponds to the factory exported by `@nextrap/nte-core`, currently `nextrap_element()`

So when working in existing code, follow the current implementation/export name, but keep the convention in mind: `nte-*` elements always use the shared `nte-element()`/`nextrap_element()` base factory.

## Import

```ts
import { nextrap_element } from '@nextrap/nte-core';
```

## Feature source: `@trunkjs/browser-utils`

`@nextrap/nte-core` is mainly a composition layer over `@trunkjs/browser-utils` mixins.

Reference package:
- `/opt/node_modules/@trunkjs/browser-utils/.ai-usage-info.md`

Prefer the helpers and mixins from `@trunkjs/browser-utils` over ad-hoc implementations for:
- logging
- event bindings
- breakpoint handling
- slot visibility
- loader / ready handling
- timing helpers like `sleep`, `waitFor`, `Debouncer`

## Supported base features

```ts
{
  logging?: boolean;
  slotVisibility?: boolean;
  eventBinding?: boolean;
  breakpoints?: boolean;
}
```

Current defaults:

```ts
{
  logging: true,
  slotVisibility: false,
  eventBinding: false,
  breakpoints: false,
}
```

## Common examples

### Minimal `nte-*` element

```ts
import { LitElement, html, unsafeCSS } from 'lit';
import { customElement } from 'lit/decorators.js';
import { nextrap_element } from '@nextrap/nte-core';
import '@nextrap/style-base';
import style from './nte-demo.scss?inline';

@customElement('nte-demo')
export class NteDemo extends nextrap_element() {
  static override styles = [unsafeCSS(style)];

  override render() {
    return html`<slot></slot>`;
  }
}
```

### With slot visibility

```ts
import { html } from 'lit';
import { customElement } from 'lit/decorators.js';
import { nextrap_element } from '@nextrap/nte-core';

const features = {
    logging: true,
    slotVisibility: false,
    eventBinding: false,
    breakpoints: false
};

@customElement('nte-dialog-like')
export class NteDialogLike extends nextrap_element(features) {
  override render() {
    return html`
      <slot name="title"></slot>
      <slot></slot>
    `;
  }
}
```

### With event bindings from `@trunkjs/browser-utils`

Use this instead of registering event listeners manually in `connectedCallback` – the mixin handles automatic cleanup and supports multiple listeners with the same event.

```ts
import { html } from 'lit';
import { customElement } from 'lit/decorators.js';
import { Listen } from '@trunkjs/browser-utils';
import { nextrap_element } from '@nextrap/nte-core';

@customElement('nte-resize-aware')
export class NteResizeAware extends nextrap_element({ eventBinding: true }) {
  override render() {
    return html`<slot></slot>`;
  }

  @Listen('resize', { target: 'window' })
  onResize() {
    this.requestUpdate();
  }
}
```

### With breakpoint support

```ts
import { customElement } from 'lit/decorators.js';
import { nextrap_element } from '@nextrap/nte-core';

@customElement('nte-responsive-box')
export class NteResponsiveBox extends nextrap_element({ breakpoints: true }) {}
```

## Agent hints

- For every new `nte-*` component, start from `nextrap_element(...)` instead of extending `LitElement` directly.
- Reuse `@trunkjs/browser-utils` features before writing custom browser helper code.
- Keep feature flags minimal: only enable the mixins actually needed.
- For shadow DOM styling, use `@nextrap/style-reset` in component SCSS and `@nextrap/style-base` only in TS as global variable bootstrap.
