# @nextrap/ntl-card

Card layout web component with optional header, image, content, footer and link slot handling.

## Import

```ts
import '@nextrap/ntl-card';
```

For styling:

```scss
@use '@nextrap/ntl-card' as card;
```

## Usage

```html
<ntl-card class="img-overlay">
  <h3 slot="header">Title</h3>
  <img slot="image" src="/image.jpg" alt="">
  <p>Card content</p>
  <p slot="footer">Footer</p>
</ntl-card>
```

Direct images or `p:has(img:not(.keep))` can be assigned to the `image` slot. Use `--aspect-ratio` or `--image-aspect-ratio` on `ntl-card` to control the image area ratio. A hidden `link` slot can make the whole card clickable when it contains an empty `a[href]`.

## Parts

Use `::part(wrapper)`, `::part(header)`, `::part(image)`, `::part(gradient)`, `::part(content)`, `::part(footer)` and `::part(link)` for Light DOM/theme styling.

## Mixins

```scss
@include card.default-style(
  $innerPadding: var(--nt-space-3),
  $border: 1px solid var(--nt-primary),
  $background-color: unset,
  $border-radius: var(--nt-border-radius),
  $image-aspect-ratio: 16 / 9,
  $overlay-background: linear-gradient(to top, rgba(0, 0, 0, 0.55), transparent),
  $modifierClasses: true
);
@include card.with-image-fullsize();
@include card.with-image-overlay();
@include card.with-modifier-classes();
```

`default-style()` includes the `.img-fullsize` and `.img-overlay` modifier classes by default. Demo/theme SCSS binds it to `ntl-card.style-default`; examples do not add a `.default` class. It does not set a wrapper gap; spacing between card sections should be handled by padding/content styles or by the parent layout. Keep Shadow DOM styles functional only; visual card styles belong in these mixins.
