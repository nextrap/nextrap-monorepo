# @nextrap/ntl-consent-blocker

Consent-gate layout component for embedding external content only after user approval.

## Import

```ts
import '@nextrap/ntl-consent-blocker';
```

For styling:

```scss
@use '@nextrap/ntl-consent-blocker' as consentBlocker;
```

## Usage

```md
---
{: layout="ntl-consent-blocker.default"}

![Map preview](/map-preview.jpg)

<template>
  <iframe title="Map" src="..."></iframe>
</template>

<button slot="pre-consent" data-action="consent">Karte laden</button>
```

- Direct `.background` children or direct `p:has(img:not(.keep))` children are assigned to the `background` slot.
- Direct `<template>` content is cloned into `consented-content` after a click on `[data-action="consent"]`.
- If background or pre-consent content is missing, `default-style()` can provide defaults via `--default-bg` and `--default-pre-consent`.

## Ausnahme von der Regel

Der ntl-consent-wrapper sollte auch in Markdown-dateien als HTML eingebunden werden. Optimalerweise werden pre-consent
immer per css gesetzt. Dies betrifft jedoch nur das element "<ntl-consent-blocker>" selbst.

## Parts

Use `::part(wrapper)`, `::part(background)`, `::part(pre-consent)`, `::part(consented-content)` and `::part(loading-text)` for theme styling.

## Mixins

```scss
@include consentBlocker.default-style();
@include consentBlocker.pairing-ntl-2col-in-ntl-consent-blocker($top-aspect-ratio: 21 / 9);
```

`default-style()` provides the default preview/pre-consent values and basic Light DOM styling for the assigned preview image. Keep Shadow DOM styles functional only; visual styles belong in package SCSS mixins.

## Pairing

See `demo/pairing-ntl-2col.md` for examples where `ntl-consent-blocker` is placed in `ntl-2col` as `.aside` and `.top` content.
