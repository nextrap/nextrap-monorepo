# @nextrap/ntl-accordion

Accordion layout web component for embedded FAQ/detail sections.

## Import

```ts
import '@nextrap/ntl-accordion';
```

For styling:

```scss
@use '@nextrap/ntl-accordion' as accordion;
```

## Usage

```html
<ntl-accordion class="default" initial-open-index="0" marker-position="end">
  <section>
    <h3>Question</h3>
    <p>Answer text.</p>
  </section>
</ntl-accordion>
```

In Markdown/content-pane layouts, put `layout="ntl-accordion.default"` on the parent section; direct child sections become `ntl-accordion-item`.

## Behavior

- `ntl-accordion` converts direct `section:not(.keep)` children to `ntl-accordion-item` via `SubLayoutApplyMixin`.
- Each item moves its first direct heading into the `title` slot.
- `initial-open-index` opens one item on first render (`initial-open-index` without value means `0`).
- `exclusive` closes other items when a new item opens.
- `marker-position="start|end"` and `marker-icon="chevron|plus"` on the accordion are propagated to items unless an item already defines the attribute.
- Accordions are usually embedded in parent layouts/cards; do not add external margins in the component or default mixin.

## Parts

Accordion: `accordion`.

Item: `details`, `summary`, `title`, `marker`, `content`.

## Mixins

```scss
@include accordion.default-style(
  $item-gap: 0,
  $border: 1px solid var(--nt-border-color, #e5e7eb),
  $border-radius: var(--nt-border-radius),
  $summary-padding: var(--nt-space-3),
  $content-padding: var(--nt-space-3),
  $summary-background: var(--nt-light-subtle),
  $summary-background-hover: var(--nt-light),
  $summary-color: inherit,
  $content-background: transparent,
  $marker-size: 1.25rem,
  $marker-color: currentColor,
  $focus-outline: 2px solid var(--nt-focus-ring-color, var(--nt-primary)),
  $modifierClasses: true
);
@include accordion.with-details-end();
@include accordion.with-marker-start();
@include accordion.with-marker-plus();
@include accordion.with-modifier-classes();
```

`default-style()` includes the `.details-end`, `.marker-start` and `.marker-plus` modifier classes by default. Visual styling belongs in these mixins or theme SCSS; Shadow DOM CSS should stay functional only.
