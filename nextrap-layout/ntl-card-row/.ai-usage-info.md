# @nextrap/ntl-card-row

Responsive card-row layout web component. Direct content sections are automatically rendered as `ntl-card` children.

## Import

```ts
import '@nextrap/ntl-card-row';
```

For styling:

```scss
@use '@nextrap/ntl-card-row' as cardRow;
```

The package publishes its root/index SCSS and `src/scss/**/*.scss` assets so themes can import these mixins.

## Usage

```html
<ntl-card-row class="default" style="--default-cols: 3">
  <h2 slot="header">Cards</h2>
  <ntl-card>Card 1</ntl-card>
  <ntl-card class="highlight" style="--cols: 6">Card 2</ntl-card>
  <ntl-card>Card 3</ntl-card>
</ntl-card-row>
```

Markdown/content-pane example:

```md
## Cards
{: layout="ntl-card-row.with-borderless-cards" section-style="--default-cols: 4;"}

### Optional row header
{: slot="header"}

### Card 1

Text

### Card 2
{: section-style="--cols: 6" .highlight}

Wide highlighted card
```

Demo modifier classes should be defined on `ntl-card-row` in demo/theme SCSS and include mixins there:

```scss
ntl-card-row.with-borderless-cards {
  @include cardRow.with-borderless-cards();
}

ntl-card-row.with-item-separators {
  @include cardRow.with-item-separators($separator-color: var(--nt-gray-400));
}
```

## Behavior

- Uses `BreakPointMixin`; the current mode is reflected as `mode="mobile"` or `mode="desktop"`.
- Uses `SubLayoutApplyMixin` to assign direct headings to `header`, direct `footer` to `footer`, and direct `section` children to `ntl-card`.
- Desktop layout uses a 12-column model. Set `--default-cols` on `ntl-card-row` and optional `--cols` on individual cards.
- Mobile mode stacks all cards full-width.

Important CSS variables:

- `--breakpoint`: layout switch breakpoint, default `xl`
- `--container-width`: row container width, default `var(--nt-container-width, 100%)`
- `--default-cols`: default desktop card width in a 12-column model
- `--cols`: per-card desktop width override
- `--gutter-x` / `--gutter-y`: card gaps

## CSS parts

Use `::part(container)`, `::part(header)`, `::part(main)` and `::part(footer)` for Light DOM/theme styling.

## Mixins

```scss
@include cardRow.default-style(
  $container-width: var(--nt-container-width, 100%),
  $default-cols: 6,
  $gap: var(--nt-gap),
  $inner-padding: 0,
  $container-background: transparent,
  $container-border: none,
  $container-border-radius: 0,
  $modifierClasses: true
);

@include cardRow.with-borderless-cards();
@include cardRow.with-footer-separators($separator-color: var(--nt-primary));
@include cardRow.with-item-separators($separator-color: var(--nt-gray-400));
@include cardRow.with-raised-card();
@include cardRow.with-modifier-classes();
```

`default-style()` includes modifier classes by default. Available classes: `.highlight` for raised cards and `.footer-separators` / legacy `.style-footer` for desktop footer separators.

- `with-borderless-cards()` sets `--border: none` and `--inner-padding: 0` for all direct `ntl-card` children.
- `with-item-separators()` adds separator pseudo-elements between direct `ntl-card` children: vertical in desktop mode at half the horizontal gap, horizontal in mobile/tablet mode at half the vertical gap.

Keep Shadow DOM styles functional only; visual row styles belong in these mixins.
