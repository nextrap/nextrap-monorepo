# @nextrap/ntl-2col

## Purpose

Responsive two-column layout web component for Nextrap content sections.

It renders one main content column and one optional `aside` column. On mobile it stacks the columns; on desktop it uses a 12-column width model controlled through CSS variables.

## Import

```ts
import '@nextrap/ntl-2col';
```

For styles / mixins:

```scss
@use '@nextrap/ntl-2col' as twoCol;

ntl-2col.default {
  @include twoCol.default-style();
  @include twoCol.<feature>();
}
```

## Slots

Shadow DOM structure:

```html
<ntl-2col>
  <div slot="top">Optional top content</div>
  Main content
  <p slot="aside">Aside content</p>
  <div slot="bottom">Optional bottom content</div>
</ntl-2col>
```

or in Markdown:

```markdown

## Some header
{: layout="ntl-2col"}

![](https://placehold.co/600x400?text=Hello+World)

Main Content
```

The component uses `SubLayoutApplyMixin` data queries:

```html
<slot name="top" data-query=":scope > .top"></slot>
<slot name="aside" data-query=":scope > .aside | :scope > p:has(img)" data-set-attribute-class="auto"></slot>
<slot name="bottom" data-query=":scope > .bottom"></slot>
```

That means direct `.top`, `.bottom`, `.aside` children are assigned automatically. A direct `p:has(img)` is assigned to `aside` and receives class `auto` if no explicit slot was set.

## Common usage examples

### Basic two-column layout

```html
<ntl-2col class="default" style="--cols: 8">
  <h2>Main title</h2>
  <p>Main text content.</p>

  <p slot="aside">
    <img src="/image.jpg" alt="Example" />
  </p>
</ntl-2col>
```

### Automatic image aside from markdown/content-pane output

When content processing creates a direct paragraph with an image, `ntl-2col` can automatically move it into the `aside` slot:

```html
<ntl-2col class="default">
  <h2>Content</h2>
  <p>Lorem ipsum...</p>
  <p><img src="/image.jpg" alt="Example" /></p>
</ntl-2col>
```

After layout processing this behaves like:

```html
<ntl-2col class="default">
  <h2>Content</h2>
  <p>Lorem ipsum...</p>
  <p slot="aside" class="auto"><img src="/image.jpg" alt="Example" /></p>
</ntl-2col>
```

The `default-style()` mixin contains the image object-fit handling for `p[slot='aside'].auto`.

### Mobile image-first behavior

The mobile image-first behavior is implemented in pure CSS in `default-style()` using `:has()` and `::part(...)`:

```scss
&[mode='mobile']:has(> p[slot='aside'].auto),
&[mode='mobile']:has(> [slot='aside'] p.auto) {
  &::part(main) {
    order: 2;
  }

  &::part(aside) {
    order: 1;
  }
}
```

Do not try to implement this with Shadow DOM `::slotted(...)`: `::slotted()` can only style the assigned Light DOM node itself and cannot select back into `#main`, `#aside` or `#wrapper`.

### Explicit top / bottom content

```html
<ntl-2col class="default">
  <div slot="top">Intro above both columns</div>

  <p>Main content</p>
  <aside slot="aside">Aside content</aside>

  <div slot="bottom">Outro below both columns</div>
</ntl-2col>
```

## Responsive behavior

The component uses the `BreakPointMixin`; the current mode is reflected as a `mode` attribute, for example:

```html
<ntl-2col mode="mobile"></ntl-2col>
<ntl-2col mode="desktop"></ntl-2col>
```

Important CSS variables:

- `--breakpoint`: breakpoint where the layout switches; default is `md`
- `--cols`: main column width in a 12-column grid; default is `6`
- `--container-width`: container width; default is `var(--nt-container-width, 100%)`
- `--gap`: gap between columns; default is `var(--nt-gap)`

On desktop `--cols` defines the main column width. The aside column is flex-based and always takes the remaining available width:

```scss
--main-width: calc(100% * var(--cols, 1) / 12);
#main { flex: 0 0 var(--main-width); }
#aside { flex: 1 1 0; }
```

Example:

```html
<ntl-2col class="default" style="--cols: 8; --breakpoint: lg">
  <p>Main column is 8/12 on desktop.</p>
  <p slot="aside">Aside is 4/12 on desktop.</p>
</ntl-2col>
```

## CSS parts

Use exported parts for external styling:

- `container`
- `top`
- `wrapper`
- `main`
- `aside`
- `bottom`

Example:

```scss
ntl-2col.hero::part(wrapper) {
  align-items: center;
}

ntl-2col.hero::part(aside) {
  min-height: 20rem;
}
```

## Styling mixins

The package exposes SCSS mixins from `index.scss`.

Available mixins:

```scss
@include default-style(
  $padding-x: var(--nt-text-gap),
  $padding-y: var(--nt-text-gap),
  $gap: var(--nt-gap),
  $background: var(--nt-light),
  $border: 1px solid var(--nt-dark),
  $border-radius: var(--nt-border-radius),
  $objectFit: yes,
  $aspectRatio: none
);

@include reverse;
@include alternating;
@include breakout-end;
@include breakout-start;
```

### Default style

```scss
@use '@nextrap/ntl-2col' as twoCol;

ntl-2col.default {
  @include twoCol.default-style();
}
```

`default-style()` handles:

- container background, border and border radius
- wrapper gap and background
- main column padding
- optional aside aspect ratio
- automatic image sizing for `p[slot='aside'].auto > img`
- mobile image-first ordering for automatic aside images

### Alternating sections

Use `alternating` on a parent/container to reverse every even `ntl-2col` on desktop:

```scss
@use '@nextrap/ntl-2col' as twoCol;

.alternating {
  @include twoCol.alternating();
}
```

Example markup:

```html
<section class="alternating">
  <ntl-2col class="default">...</ntl-2col>
  <ntl-2col class="default">...</ntl-2col>
</section>
```

### Breakout variants

Breakout is CSS-only. The Shadow DOM container exposes configurable margins (`--container-margin-left` / `--container-margin-right`) and a separate render width (`--container-render-width`) so mixins can extend the clipped container to one viewport edge while keeping the text column, top slot and bottom slot inside the configured `--container-width`.

```scss
@use '@nextrap/ntl-2col' as twoCol;

ntl-2col.default.breakout-end {
  @include twoCol.default-style();
  @include twoCol.breakout-end(); // aside/image extends to the right viewport edge
}

ntl-2col.default.breakout-start {
  @include twoCol.default-style();
  @include twoCol.breakout-start(); // aside/image extends to the left viewport edge
}
```

## Agent hints

- Keep Shadow DOM CSS minimal and functional only.
- Put visual styling into SCSS mixins using `::part(...)`.
- Do not import `@nextrap/style-base` into Shadow DOM styles; use global `--nt-*` variables from outside.
- Use `@nextrap/style-reset` in the component styles array for baseline Shadow DOM reset.
- For nested slot/content processing, reuse the existing `data-query` / `SubLayoutApplyMixin` pattern.
- Use `:has()` in Light DOM styling when parent state depends on slotted Light DOM children.
- Do not use `::part(...)` inside Shadow DOM stylesheets.
- `::slotted(...)` is only suitable for styling the slotted element itself, not for changing sibling Shadow DOM layout.
